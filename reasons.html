Pure-Functional Language

Unlike some other functional programming language Haskell is pure. Meaning, it doesnot allow any side-effects. A side effect would mean that evaluating an expression changes some internal state that would later cause evaluating the same expression to have a different result. In a pure functional language we can evaluate the same expression as many times with the same arguments, and it would always return the same value, because there is no state change. Because functional language are more intuitive and offer more and easier ways to get the job done, functional programming tend to be shorter.

Lazy Programming Language

Haskell is called lazy programming language because of it's lazy evluation nature. Lazy evaluation is also know by the name of call-by-need. Lazy evaluation means the evalution technique which delays the evalution of an expression untill its vallue is needed.
