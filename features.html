Haskell is purely-functional.

What does it mean for a program to be purely-functional? A purely functional programming language is one that executes its code using expressions, rather than statements which can affect the global state of a program.

Functional programming uses higher-order functions first-class, meaning they are treated like any other value and can be passed into other functions, as well as be the resulting return of a function. Functions that take other functions as paramters are called higher-order functions, and are very useful when creating loops, and code refactoring.
